This principles would allow us to create apis that are:
	1. Consistent
	2. Simple
	3. Secure
	4. Performant

The best api is the one that we can use without reading the documentation

---

**The GOOD API should be**

1. Consistent:
	- Consistent naming, casing and patterns
2. Simple:
	- Keep it simple
	- Focus on core use cases
	- Intuitive design, reduce complexity
3. Secure:
	- Authentication
	- Authorization
	- Input validation
	- Rate Limiting
4. Performance:
	- Efficiency with appropriate caching strategies
	- Pagination with limits and offsets
	- Payloads should be minimal
	- Reduce the round trips
